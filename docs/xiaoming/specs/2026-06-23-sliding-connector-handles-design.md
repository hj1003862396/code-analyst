# 三段式滑动连线手柄设计文档

本文档定义了在代码调用链分析图中，解决蓝色连线拖拽手柄偏移 Bug，并实现三段式连线中首段/尾段连接点沿节点边缘滑动、中段水平移动的正交连线拖拽交互方案。

## 1. 需求背景与目标

1. **手柄位置正确**：解决蓝色手柄偏离连线、出现在画布左上角的渲染 Bug，确保手柄始终贴合在连线各段中点。
2. **三段式滑动拖拽**：
   - 拖拽首段水平线：连线保持正交，且起点（首点）能够沿着源节点右边缘上下滑动。
   - 拖拽中段垂直线：水平移动垂直线段，调整折弯的横向位置。
   - 拖拽尾段水平线：连线保持正交，且终点（尾点）能够沿着目标节点左边缘上下滑动。
3. **拖拽后流畅更新**：拖拽时手柄跟随鼠标移动，节点被拖动或展开时，连线和手柄实时自适应重绘。

---

## 2. 方案设计

### 2.1 节点多锚点定义
为了让连线起点和终点可以平滑滑动，我们将原来固定的 2 个锚点扩展为左右边缘各 21 个细粒度锚点（滑动步长为 5%）：
- **左侧边缘（连接终点）**：坐标从 `[0, 0]` 到 `[0, 1]`。索引号为 `0` - `20`。
- **右侧边缘（连接起点）**：坐标从 `[1, 0]` 到 `[1, 1]`。索引号为 `21` - `41`。

在自定义节点 `custom-node` 的 `getAnchorPoints` 中返回这些点。

### 2.2 修复路径解析与手柄实时更新
1. **统一路径格式**：在 `parsePathPoints` 中，如果传入的 `path` 是数组，则先通过 `join` 转换为 SVG 格式字符串，避免 `A` (弧线) 命令参数解析错位。
2. **实现 `afterUpdate`**：在自定义边 `custom-polyline` 中，同时在 `afterDraw` 和 `afterUpdate` 调用 `drawHandles` 绘制/更新手柄。每次绘制前，先根据 `name` 属性清除旧手柄，并基于最新路径重新绘制手柄，确保手柄随连线移动。

### 2.3 重构 `edge:mousedown` 拖拽行为
监听拖拽的 handle 索引：
- **`index === 0` (首段)**：更新 `cps[0].y = point.y`，并计算 `sourceAnchor` 在源节点右边缘的比例，更新 `sourceAnchor` 索引。
- **`index === 1` (中段)**：水平移动中段，将 `cps[0].x` 和 `cps[1].x` 更新为 `point.x`。
- **`index === 2` (尾段)**：更新 `cps[1].y = point.y`，并计算 `targetAnchor` 在目标节点左边缘的比例，更新 `targetAnchor` 索引。

5. 修改 `edge:mousedown` 的拖拽事件响应，实现滑动比例计算与锚点实时更新。

## 4. 验证方案 (无需开启页面，确认逻辑无误)

### 4.1 核心数据结构与公式校验
1. 初始边创建默认值：右侧中心 (索引 31) 和左侧中心 (索引 10) 是否匹配 `[1, 0.5]` 和 `[0, 0.5]`。
   - 左侧：21 个点，比例从 0 到 1。中间点比例 0.5，对应索引为 `0 + 0.5 * 20 = 10`。正确。
   - 右侧：21 个点，比例从 0 到 1。中间点比例 0.5，对应索引为 `21 + 0.5 * 20 = 31`。正确。
2. 滑动比例与索引换算：
   - 首段拖拽计算：`sourceRatio = (point.y - sourceModel.y + sourceHeight / 2) / sourceHeight`。
   - `newSourceAnchor = 21 + Math.round(sourceRatio * 20)`。
   - 当 `point.y = sourceModel.y` 时，`sourceRatio = 0.5`，`newSourceAnchor = 21 + 10 = 31`。正确。
