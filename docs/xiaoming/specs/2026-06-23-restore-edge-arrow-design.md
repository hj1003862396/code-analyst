# 连线尾部箭头恢复设计文档

本文档定义了在代码调用链分析工具中，恢复关系线尾部指向目标的箭头，并保证其能随着三段式手柄的调整和节点拖拽自动更新指向的设计方案。

## 需求背景
1. **指示流向**：调用链中的关系线需要箭头来明确指出方法之间的调用流向（由调用方指向被调用方）。
2. **手柄联动**：在手动拖拽连线的三段式手柄调整弯折形状时，或者在拖动节点位置时，箭头需要能够自动且平滑地跟随连线终点并精确指向节点锚点。

## 方案设计

### 1. 修改 G6 Graph 的 `defaultEdge` 样式配置
在 `src/main/resources/static/index.js` 的 `initGraph` 函数中，为 `defaultEdge.style` 增加 `endArrow` 属性：
- 使用 G6 的 `G6.Arrow.triangle` 绘制标准三角形。
- 箭头尺寸设置：宽度为 8px，长度为 10px。
- 填充颜色（`fill`）和描边颜色（`stroke`）设置为与线条相同的黑色（`#000000`）。
- 偏移量 `d` 设置为 0，使箭头尖端刚好触及节点的接入锚点。

## 模块修改清单

### `src/main/resources/static/index.js`
- **修改位置**: 在 `initGraph` 的 `defaultEdge` 样式中，添加 `endArrow` 配置。

## 验证方案

### 1. 编译与启动测试
- 执行 `mvn clean compile` 确认编译无误。
- 执行 `mvn spring-boot:run` 启动服务。

### 2. 功能性验证
- 打开分析工具页面（`http://localhost:8080`），输入入口类和方法，加载调用链。
- 展开子方法，确认生成的每一条圆角折线尾部都带有指向目标节点的黑色三角形箭头。
- 拖动折线中段或前后的蓝色胶囊形手柄，确认箭头能自动且准确地贴合目标节点锚点并调整角度。
- 拖拽任意节点卡片，确认所有与之相连的连线及其箭头都能实时重绘和更新指向。
